Smart Reviewer — System Architecture

Next.js (App Router) · Vercel · MongoDB Atlas · GNews · OpenAI

Frontend
Next.js Client Components — runs in the browser
🔍 Search Panel
Text input + search button. Calls backend to fetch articles from GNews.
📰 Article List
Displays returned articles. Each has an "Analyze" button to trigger AI review.
🤖 Analysis View
Shows AI summary + sentiment badge (positive / neutral / negative) for selected article.
📋 Results Table
Persistent table of all previously analyzed articles pulled from MongoDB.
REST / JSON
Backend
Next.js Route Handlers — runs on Vercel
GET /api/news
Proxies search query to GNews. Keeps API key server-side.
POST /api/analyze
Accepts article. Calls OpenAI once for both summary + sentiment. Saves to MongoDB.
GET /api/results
Returns all analyzed articles from MongoDB for the results table.
News fetch
1 AI call
Read / Write
External
External APIs
📡 GNews.io
Free tier: 100 req/day. Returns recent articles by keyword.
🧠 OpenAI GPT-4o-mini
Single structured prompt returns both summary and sentiment. Key already in .env.
Database
🍃 MongoDB Atlas
Free-tier cloud cluster. Stores analyzed article reviews.

User Flows

1
Search for articles
User types a topic → React calls GET /api/news?q=topic → Route handler calls GNews.io → Returns list of articles → React renders the list
2
Analyze an article
User clicks "Analyze" → React calls POST /api/analyze with article data → Route handler checks MongoDB (skip AI if already cached) → If new: single OpenAI call returns { summary, sentiment, confidence } → Saved to MongoDB → Result returned to frontend → Displayed inline
3
View all stored results
React calls GET /api/results on load → Route handler queries MongoDB → Returns all stored results → Rendered in the Results Table (title, source, sentiment, summary snippet, date)
4
Loading / Error states
Every async operation shows a loading spinner → Network/API errors are caught and shown as inline error messages → No silent failures

Key Design Decisions

Sentiment Strategy
Single OpenAI call returns both summary and sentiment. smart
Prompt asks for structured JSON: { summary, sentiment, confidence }. Zero extra API calls.
Caching
Before calling OpenAI, check MongoDB by article URL. perf
If already analyzed, return stored result instantly. Saves API quota.
API Keys Security
All external API keys (OpenAI, GNews) live only on the backend. The React frontend never sees them.
Project Structure
Next.js App Router: frontend and API route handlers in one project. Deployed to Vercel via GitHub CI/CD.
